# qosnat2 2026072011


## 概要

修复独立 IP 代理出站策略：源 CIDR 经 ProxyEgress TUN 可正常上网

## 新增

- （无）

## 优化

- （无）

## 修复

- sing-box TUN 栈改为 `gvisor`：`system`/`mixed` 在 `auto_route=false` 时无法可靠接管进 TUN 的 TCP，导致策略路由到代理后无法上网（代理连通性测试仍可能显示通过）
- ProxyEgress 出站强制 MASQUERADE，不再把 TUN 地址当作 SNAT 源地址
- 托管 WanLink（WARP/Proxy）不再生成 WAN 入站默认丢弃规则，避免误伤 TUN/veth 回程
- HTTP(S) 代理将 UDP（含 DNS）直连，避免不支持 UDP 的代理导致解析失败
- 代理/WARP 断开时 WanLink 为 disabled，引用该链路的出站策略保留且不再报「wan link not found or disabled」

- （无）

## 其他

- 升级后请重新连接独立 IP 代理，使新配置（gvisor / UDP 直连）生效；并确认出站策略已绑定对应代理 WanLink
