## 5. NAT 验证（客户端临时 /32 路由）

### 5.1 方法

```bash
# 客户端 100.64.0.254
ip route add 1.1.1.1/32 via 100.64.0.249 dev eth1
ping -c 4 -I eth1 1.1.1.1
# 测试后回收
ip route del 1.1.1.1/32 via 100.64.0.249 dev eth1
```

### 5.2 结果

| 项 | 结果 |
|----|------|
| 路由切换 | `ip route get 1.1.1.1` → `via 100.64.0.249 dev eth1 src 100.64.0.254` |
| ping 回包 | 无上游回包（预期，外网不可达） |
| **conntrack SNAT** | `src=100.64.0.254 dst=1.1.1.1` → 回复侧 `dst=203.0.113.10` |
| 路由回收 | `ip route del` 后恢复走 `eth0` 默认路径 |

结论：**策略 SNAT 到共享 IP `203.0.113.10` 已生效**（`100.64.0.254/32` 在 policy_routes 中）。

### 5.3 NAT 与 QoS 合路

iperf 客户端→WAN 时 conntrack 可见 `100.64.0.254 ↔ 157.15.107.249:5201` 会话；与 QoS 整形同路径。
