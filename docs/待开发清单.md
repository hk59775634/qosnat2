## 需实机验收（§14）

在双网卡网关环境由人工执行并勾选：

- [ ] 内网单 /32 **iperf** 相对模板速率 **±5%**
- [ ] **VIP** /32 覆盖同网段默认 profile 速率
- [ ] **24h** NAT + QoS 联跑无 conntrack 打满、无 map 泄漏
- [ ] **重启** `qosnatd` 后 eBPF Map 条目与 `state.json` 一致（profile + host_exact）
- [ ] GeoIP 规则 + 样例 CIDR 文件生效（WAN 入向 drop）
- [ ] 多 WAN 两条 default（不同 metric） failover 符合 Tier/Metric 预期

辅助命令：

```bash
sudo /opt/qosnat2/scripts/acceptance-check.sh
# 已登录或 export QOSNAT_API_KEY=...
/opt/qosnat2/scripts/test-ui-api.sh
```

## HTTPS 启用示例

```bash
openssl req -x509 -newkey rsa:2048 -keyout /etc/qosnat2/tls.key -out /etc/qosnat2/tls.crt -days 365 -nodes -subj /CN=qosnat2

# /etc/qosnat2/env 增加：
TLS_CERT=/etc/qosnat2/tls.crt
TLS_KEY=/etc/qosnat2/tls.key

systemctl restart qosnatd
```

## 已知限制（维护时勿忘）

- 鉴权：HTTP(S) Session Cookie + `X-API-Key`，无 RBAC / MFA
- Aliases / GeoIP set：仅 `ipv4_addr` interval
- 防火墙规则模型简化；复杂策略编辑 `/etc/qosnat2/nftables-qosnat.nft` 后勿被 `Apply` 覆盖（当前每次 Apply 会重写该文件）
- GeoIP 无数据文件时对应国家规则不写入 nft（无 UI 错误提示）
- 多 WAN **Weight** 字段当前为占位，不参与内核选路
- VPN 仅 WireGuard；不支持 IPsec / OpenVPN
